# HTML Video Bitrate Viewer

A client-side, lightweight tool designed to visualize video bitrate data generated by FFprobe. It's ideal for scenarios where software installation is restricted, as it runs directly in your web browser.

## Overview

This project offers two distinct viewers, each tailored for different levels of detail and performance:
1.  **Frame-based Viewer (`frame.html`):** Provides detailed per-frame bitrate analysis, including I/P/B frame types.
2.  **Packet-based Viewer (`packet.html`):** Offers a faster, more general bitrate overview based on packet sizes, without frame type information.

Both viewers are built to be interactive, allowing users to explore bitrate trends, zoom, pan, and adjust smoothing.

## Features (Common to both viewers)

*   **Client-Side Operation:** Runs entirely in your web browser; no server or software installation required.
*   **Interactive Chart:** Visualize bitrate over time with zoom, pan, and unit switching (`kbps` / `Mbps`).
*   **Sliding Window Smoothing:** Apply smoothing to reveal clearer bitrate trends.
*   **Drag-and-Drop Interface:** Easily load FFprobe JSON output files.

---

## 1. Frame-based Bitrate Viewer (`frame.html`)

This viewer provides a granular analysis of your video's bitrate, showing the size of individual frames and their types (I, P, B). It's perfect for in-depth video analysis and understanding encoding efficiency.

### Key Characteristics

*   **Detailed Analysis:** Visualizes bitrate for each individual video frame.
*   **Frame Type Markers:** Clearly distinguishes between I-frames (intra-coded), P-frames (predictive), and B-frames (bi-directional predictive) on the chart.
*   **Data Source:** Requires a `frames.json` file generated with FFprobe's `-show_frames` option.

### Usage

1.  **Generate `frames.json`:**
    Use FFprobe to extract frame-level metadata from your video file. The `-show_frames` option is crucial for this detailed output.

    ```bash
    ffprobe -v quiet -select_streams v:0 -show_frames -show_entries frame=best_effort_timestamp_time,pkt_size,pict_type -of json input.mp4 > frames.json
    ```
    *Note: Generating frame-level data requires more processing from FFprobe as it involves partial decoding of the video stream.*

2.  **Open the Viewer:**
    Open the `frame.html` file in your web browser.

3.  **Load Data:**
    Drag and drop the generated `frames.json` file onto the viewer page.

4.  **Explore:**
    Interact with the chart to zoom, pan, adjust smoothing, and toggle the visibility of different frame types.

---

## 2. Packet-based Bitrate Viewer (`packet.html`)

For a quicker overview of bitrate distribution without the need for detailed frame type information, the packet-based viewer is more efficient. It processes packet sizes directly, which is faster for FFprobe to generate.

### Key Characteristics

*   **Faster Data Generation:** FFprobe can generate packet-level data (`-show_entries packet`) significantly faster than frame-level data, as it doesn't require full video decoding.
*   **General Bitrate Overview:** Provides a good representation of bitrate trends based on packet sizes.
*   **No Frame Type Information:** This viewer does not display I/P/B frame markers.
*   **Data Source:** Requires a `result.json` file generated with FFprobe's `-show_entries packet` option.

### Usage

1.  **Generate `result.json`:**
    Use FFprobe to extract packet-level metadata from your video file.

    ```bash
    ffprobe -v quiet -select_streams v:0 -show_entries packet -of json input.mp4 > result.json
    ```
    *Note: This command is generally much faster than the one for `frames.json` because it avoids full video frame decoding.*

2.  **Open the Viewer:**
    Open the `packet.html` file in your web browser.

3.  **Load Data:**
    Drag and drop the generated `result.json` file onto the viewer page.

4.  **Explore:**
    Interact with the chart to zoom, pan, and adjust smoothing.
